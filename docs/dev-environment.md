## Dev environment: Conductor + portless

- `scripts/setup.sh` is the Conductor workspace setup script — symlinks `.env` files from `$CONDUCTOR_ROOT_PATH` (for `apps/web`, `apps/dashboard`, `packages/backend`, and `packages/backend/convex`) and runs `bun install`. If you add another frontend app, add its symlink line here too.
- `scripts/dev.sh` simply runs `bun run dev`.
- Both frontend apps (`apps/web` and `apps/dashboard`) use `portless run vite dev` — [portless](https://github.com/nicepkg/portless) assigns stable named localhost URLs per app, derived from each workspace's package name (e.g. `https://web.localhost` and `https://dashboard.localhost` here, plus worktree-specific subdomains automatically). Better Auth's `trustedOrigins` in dev includes both `http://localhost:*` and `https://*.localhost` to support portless HTTPS for either app.
- **The portless proxy daemon must be running before `bun dev` URLs work.** It binds to port 443, doesn't survive a reboot, and the auto-start path silently fails when sudo isn't available — so after every reboot, run `sudo portless proxy start` once before `bun dev`. `portless run` will print "Proxy is running" and register routes either way (those are persisted on disk), but if nothing's bound to 443 (`lsof -nP -iTCP:443 -sTCP:LISTEN` empty), `https://*.localhost` will refuse the connection. The local CA installed on first run does persist via the system keychain, so no repeat trust prompt.
- The shared Vite config in `packages/vite-preset/src/index.ts` sets `eventBusConfig: { port: 0 }` on the TanStack Start devtools plugin to avoid `EADDRINUSE` when running multiple worktrees concurrently. Both apps' `vite.config.ts` is a 3-line delegation to `createAppConfig()` — edit the preset, not the app file.
